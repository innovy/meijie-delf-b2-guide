保存设置

这个密码需要你自己在 Netlify 里设置。保存时会同步 Markdown 源文件，并生成前台读取的 data.json。

第一次上线前要填什么？

进入 Netlify 项目后，找到 Site configuration → Environment variables，新增 ADMIN_PASSWORD、GITHUB_TOKEN、GITHUB_REPO、GITHUB_BRANCH。

ADMIN_PASSWORD 是你自己设置的后台密码，不是邮箱密码，也不是 GitHub 密码。GITHUB_REPO 只填“用户名/仓库名”，不要填完整网址。

首次部署保护默认开启：保存前必须先在目标仓库找到 content/data.json，避免误写到空仓库或错仓库。保存成功后会同步 Markdown 源文件、content/site.json 和生成 JSON。

线上 /admin/ 可以保存到 GitHub；首次上线后先点“检查保存链路”。本机写 Markdown 后运行构建脚本即可同步前台数据。
